# ПайсМобайл — фактическая документация API и HTTP-маршрутов

Документация составлена по фактическому коду Flask-приложения `paismobile-flask`. Важно: приложение не является чистым REST API. Большая часть маршрутов возвращает HTML-страницы или делает `302 Redirect`, а не возвращает JSON. JSON-ответы есть только у части методов, которые используются как API-подобные операции.

Базовый адрес при локальном запуске:

```text
http://127.0.0.1:5001
```

## Авторизация и формат ответов

Авторизация реализована через стандартную Flask-сессию. После успешного `POST /login` или `POST /register` в браузере появляется cookie `session`, внутри которой хранится `user_id`. Защищённые маршруты проверяют наличие `session["user_id"]`. Если пользователь не авторизован, сервер добавляет flash-сообщение и перенаправляет на:

```text
/login?next=<исходный_путь>
```

Формат JSON-ответов у API-подобных методов:

```json
{
  "ok": true,
  "message": "Текст результата"
}
```

Некоторые методы добавляют дополнительные поля, например `city` или `last4`.

## Общие правила валидации

### Имя и фамилия

Поля `first_name` и `last_name` проверяются одинаково:

- значение не должно быть пустым;
- допускаются только русские буквы `а-я`, `ё`, `А-Я`, `Ё`;
- пробелы, дефисы, латиница, цифры и спецсимволы запрещены;
- первая буква приводится к верхнему регистру.

Сообщения ошибок:

```text
Имя не может быть пустым.
Имя — только русские буквы.
Фамилия не может быть пустым.
Фамилия — только русские буквы.
```

### Телефон

Телефон нормализуется: из значения берутся только цифры. Если введён номер из 11 цифр и он начинается с `8`, первая цифра заменяется на `7`. Валидный итоговый формат:

```text
+7XXXXXXXXXX
```

Ошибка:

```text
Телефон должен быть в формате +7XXXXXXXXXX.
```

### Email

Фактическая проверка минимальная:

- значение не должно быть пустым;
- в строке должен быть символ `@`.

Домен, зона, список разрешённых почтовых сервисов и полноценный email-формат backend не проверяет.

Ошибка:

```text
Введите корректный email.
```

### Пароль

Пароль должен соответствовать правилам:

- минимум 8 символов;
- максимум 20 символов;
- минимум одна строчная латинская буква;
- минимум одна заглавная латинская буква;
- минимум один спецсимвол из набора `!@#$%^&*()_+-=[]{}|;:'",.<>?/`~№`.

Пароль не требует цифру. Кириллица не запрещена напрямую, но наличие латинских букв обязательно.

Пример ошибки:

```text
Пароль: минимум 8 символов; нет заглавной латинской буквы; нет спецсимвола.
```

### Банковская карта

Поля карты:

| Поле | Правило |
|---|---|
| `card` | 16 цифр. В `payment/topup` игнорируются любые нецифровые символы, в `save-card` удаляются пробелы и дефисы. |
| `holder` | Проверяется только непустота. Латиница и длина не проверяются. |
| `expire` | Формат `ММ/ГГ`, месяц от `01` до `12`, дата не должна быть в прошлом. |
| `cvv` | Ровно 3 цифры. |

Сообщения ошибок:

```text
Номер карты должен содержать ровно 16 цифр.
Введите имя владельца карты.
Срок действия карты должен быть в формате ММ/ГГ.
Карта просрочена.
CVV должен содержать ровно 3 цифры.
```

# Маршруты

## GET `/`

Главная страница.

**Авторизация:** не требуется.  
**Ответ:** `200 text/html`.

Возвращает главную HTML-страницу и featured-тарифы:

```text
serafim, arkhangel, prestol
```

---

## GET `/tariffs`

Страница тарифов и конструктор индивидуального тарифа.

**Авторизация:** не требуется.  
**Ответ:** `200 text/html`.

Query-параметры:

| Параметр | Тип | По умолчанию | Описание |
|---|---:|---:|---|
| `city` | string | город пользователя / город из сессии / `Москва` | Если передан, сохраняется в `session["selected_city"]`. |
| `minutes` | integer | `500` | Минуты в конструкторе. |
| `internet` | integer | `20` | ГБ в конструкторе. |
| `sms` | integer | `100` | SMS в конструкторе. |

Если `minutes`, `internet` или `sms` не приводятся к числу, используется значение по умолчанию.

Цена конструктора считается так:

```python
round(160 + minutes * 0.32 + internet * 5 + sms * 0.6)
```

Важная особенность: конструктор только считает цену на странице. Отдельного endpoint’а для подключения индивидуального тарифа нет.

---

## GET `/cities`

Страница выбора города.

**Авторизация:** не требуется.  
**Ответ:** `200 text/html`.

Query-параметры:

| Параметр | Тип | Описание |
|---|---|---|
| `q` | string | Поиск по названию города и области. Поиск регистронезависимый, как подстрока. |

Пример:

```text
/cities?q=мос
```

---

## GET `/select-city/<city>`

Выбор города через обычную ссылку.

**Авторизация:** не требуется.  
**Ответ:** `302 Redirect`.

Поведение:

1. сохраняет `city` в `session["selected_city"]`;
2. если пользователь авторизован, обновляет город пользователя в профиле;
3. добавляет flash-сообщение;
4. перенаправляет на `Referer` или `/cities`.

Особенность: значение `city` из path не проверяется по справочнику городов.

---

## PUT `/select-city/<city>`

Выбор города с JSON-ответом.

**Авторизация:** не требуется.  
**Ответ:** `200 application/json`.

Пример ответа:

```json
{
  "ok": true,
  "message": "Выбран город: Москва.",
  "city": "Москва"
}
```

## GET `/actions`

Страница акций.

**Авторизация:** не требуется.  
**Ответ:** `200 text/html`.

Возвращает список акций из `data/promos.json`.

Особенность: истёкшие акции отключены на уровне интерфейса, но backend не проверяет поле `expired`, если отправить `promo_id` вручную.

---

## POST `/actions`

Активация акции или ввод промокода.

**Авторизация:** зависит от сценария.  
**Content-Type:** `application/x-www-form-urlencoded`.  
**Ответ:** `302 Redirect`.

Поля формы:

| Поле | Описание |
|---|---|
| `promo_id` | Если передан, backend пытается активировать акцию. Требует авторизацию. |
| `promo` | Если `promo_id` пустой, любой непустой промокод считается принятым. |
| `friend_phone` | Используется для акции cashback, если форма будет отправлена вместе с promo_id cashback-акции. |

Поведение:

- если `promo_id` указан и пользователь не авторизован → redirect на `/login?next=/actions`;
- если `promo_id` указан и акция найдена → проверяются бизнес-условия акции;
- если `promo_id` не указан, но `promo` непустой → промокод принимается без проверки по справочнику;
- если оба поля пустые → flash `Введите промокод.`.

JSON не возвращается.

## POST `/login`

Вход в личный кабинет.

**Авторизация:** не требуется.  
**Content-Type:** `application/x-www-form-urlencoded`.

Поля:

| Поле | Обязательное | Описание |
|---|---|---|
| `login` | да | Email или телефон. |
| `password` | да | Пароль. |

Успех:

- записывает `session["user_id"]`;
- redirect на query-параметр `next`, если он указан;
- иначе redirect на `/my-tariff`.

Ошибка:

- возвращает HTML формы логина со статусом `200`;
- flash: `Неверный телефон/email или пароль.`.

## POST `/register`

Регистрация нового пользователя.

**Авторизация:** не требуется.  
**Content-Type:** `application/x-www-form-urlencoded`.

Поля:

| Поле | Обязательное | Правила |
|---|---|---|
| `first_name` | да | Только русские буквы. |
| `last_name` | да | Только русские буквы. |
| `phone` | да | После нормализации `+7XXXXXXXXXX`. Должен быть уникальным. |
| `email` | да | Непустой и содержит `@`. Должен быть уникальным. |
| `password` | да | 8–20 символов, строчная латинская, заглавная латинская, спецсимвол. |
| `password_confirm` | да | Должен совпадать с `password`. |
| `city` | да | Должен быть в справочнике городов. |

Успех:

- создаёт пользователя в `data/users.json`;
- записывает `session["user_id"]`;
- если указан query `next`, перенаправляет туда;
- иначе перенаправляет на `/profile`.

Ошибка:

- возвращает HTML формы со статусом `200`;
- поле с ошибкой может быть очищено в форме;
- JSON не возвращается.

## POST `/forgot`

Запрос восстановления пароля.

**Авторизация:** не требуется.  
**Content-Type:** `application/x-www-form-urlencoded`.

Поля:

| Поле | Правило |
|---|---|
| `email` | Проверяется только непустота. |

Если email непустой:

- flash: `Инструкции по восстановлению отправлены, если email есть в базе.`;
- redirect на `/login`.

Если email пустой:

- flash: `Введите email.`;
- остаётся на форме восстановления.

Реальной отправки email и генерации токена нет.

## GET `/payment/<tariff_id>`

Форма оплаты тарифа.

**Авторизация:** требуется.  
**Ответы:** `200 text/html`, `302 Redirect`, `404 text/html`.

Path-параметр:

| Параметр | Примеры |
|---|---|
| `tariff_id` | `vestnik`, `serafim`, `kheruvim`, `arkhangel`, `prestol`, `nefilim` |

- если тариф не найден → HTML 404;

- иначе возвращает форму оплаты.

## POST `/payment/<tariff_id>`

Оплата и подключение тарифа.

**Авторизация:** требуется.  
**Content-Type:** `application/x-www-form-urlencoded`.  
**Ответы:** `200 text/html` при ошибке формы, `302 Redirect` при успехе.

Поля:

| Поле | Правило |
|---|---|
| `card` | 16 цифр, в этом endpoint’е любые нецифровые символы отбрасываются. |
| `holder` | Непустое значение. |
| `expire` | `ММ/ГГ`, не в прошлом. |
| `cvv` | 3 цифры. |
| `phone` | Записывается в payload платежа, отдельно не валидируется. |
| `email` | Записывается в payload платежа, отдельно не валидируется. |

Успех:

1. создаёт запись платежа `kind = "tariff"` в `data/payments.json`;
2. сохраняет карту в профиле пользователя;
3. подключает тариф пользователю;
4. добавляет запись в историю тарифа;
5. перенаправляет на `/my-tariff`.

Особенность: визуальная форма содержит промокод, но он не отправляется и не обрабатывается backend’ом.

## POST `/topup`

Пополнение баланса.

**Авторизация:** требуется.  
**Content-Type:** `application/x-www-form-urlencoded`.

Поля:

| Поле | Правило |
|---|---|
| `amount` | Целое число больше 0. Невалидное значение превращается в `0` и даёт ошибку. |
| `card` | 16 цифр. |
| `holder` | Непустое значение. |
| `expire` | `ММ/ГГ`, не в прошлом. |
| `cvv` | 3 цифры. |
| `phone` | Записывается в payload платежа, отдельно не валидируется. |
| `email` | Записывается в payload платежа, отдельно не валидируется. |
| `promo` | Есть в HTML, но backend не обрабатывает. |

Успех:

1. создаёт запись платежа `kind = "topup"`;
2. сохраняет карту пользователю;
3. увеличивает баланс на `amount`;
4. redirect на `/my-tariff`.

## POST `/profile`

Обновление профиля через HTML-форму.

**Авторизация:** требуется.  
**Content-Type:** `application/x-www-form-urlencoded`.  
**Ответ:** `302 Redirect`.

Поля:

| Поле | Правило |
|---|---|
| `first_name` | Только русские буквы. |
| `last_name` | Только русские буквы. |
| `birth_date` | Сохраняется как строка, возраст и будущая дата не проверяются. |
| `gender` | Сохраняется как строка, допустимые значения backend не проверяет. |
| `phone` | Формат после нормализации `+7XXXXXXXXXX`; уникальность проверяется, если номер изменён. |
| `email` | Непустой и содержит `@`; уникальность проверяется, если email изменён. |
| `city` | Если не пустой, должен быть в справочнике городов. |
| `current_password` | Проверяется только если непустой и меняется пароль. |
| `new_password` | Если указан, запускает смену пароля. |
| `password_confirm` | Должен совпасть с новым паролем. |

Особенность: если `new_password` указан, а `current_password` пустой, код не проверяет старый пароль.

## PUT `/profile`

Обновление профиля с JSON-ответом.

**Авторизация:** требуется.  
**Content-Type:** `application/json` или `application/x-www-form-urlencoded`.

Тело запроса аналогично `POST /profile`.

Успешный ответ без смены пароля:

```json
{
  "ok": true,
  "message": "Настройки профиля сохранены."
}
```

Успешный ответ со сменой пароля:

```json
{
  "ok": true,
  "message": "Пароль обновлён."
}
```

Ошибка валидации:

```json
{
  "ok": false,
  "message": "Телефон должен быть в формате +7XXXXXXXXXX."
}
```

HTTP-статус ошибки: `400`.

|---:|---|---|---|

# Как подключить Swagger UI

1. Установить зависимость:

```powershell
pip install flask-swagger-ui
```

2. Добавить в `requirements.txt`:

```text
flask-swagger-ui
```

3. В `app.py` добавить импорт:

```python
from flask_swagger_ui import get_swaggerui_blueprint
```

4. После создания `app = Flask(__name__)` добавить:

```python
SWAGGER_URL = "/swagger"
API_URL = "/static/swagger.json"

swaggerui_blueprint = get_swaggerui_blueprint(
    SWAGGER_URL,
    API_URL,
    config={"app_name": "ПайсМобайл API"}
)

app.register_blueprint(swaggerui_blueprint, url_prefix=SWAGGER_URL)
```

5. Положить файл `swagger.json` в папку:

```text
static/swagger.json
```

6. Запустить приложение и открыть:

```text
http://127.0.0.1:5001/swagger
```
